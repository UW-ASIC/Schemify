The tokenizer produces tokens from Tcl source text. Key token types:
```zig
pub const Token = struct {
    tag: Tag,
    start: usize,
    end: usize,

    pub const Tag = enum {
        eof,
        newline,
        semicolon,
        whitespace,
        word,           // bare word (command name, variable name, value)
        quoted_string,  // "..." delimited string
        braced_string,  // {...} delimited string (no substitution inside)
        variable,       // $name or ${name}
        bracket_cmd,    // [...] command substitution
        comment,        // # to end of line (only valid at command position)
    };
};

pub const Tokenizer = struct {
    source: []const u8,
    pos: usize,

    pub fn init(source: []const u8) Tokenizer;
    pub fn next(self: *Tokenizer) Token;
    pub fn slice(self: *const Tokenizer, token: Token) []const u8;
};
```

Critical tokenizer behaviors:
- Brace-delimited strings `{...}` track brace depth and do NOT process escapes inside (Tcl rule: braces suppress substitution)
- Quoted strings `"..."` process `\"`, `\\`, `\n`, `\t` escapes and allow `$var` and `[cmd]` substitution inside
- Variable references `$name`, `${name}`, `$env(NAME)` are recognized as variable tokens
- Bracket commands `[cmd args]` track bracket depth for nested commands
- Comments `#` are only valid at command position (start of line or after `;`)
- Newlines and semicolons are command separators
- Backslash-newline is line continuation (consume `\\\n` and following whitespace, treat as single space)

Create `plugins/EasyImport/src/TCL/expr.zig` (~200 lines). Rewrite from scratch. Consult old TCL.zig lines 700-900 (expression evaluation) as behavioral reference.

The expression parser evaluates Tcl `expr` expressions:
```zig
pub fn evalExpr(
    source: []const u8,
    lookup_var: *const fn(name: []const u8) ?[]const u8,
    lookup_env: *const fn(name: []const u8) ?[]const u8,
) !ExprResult;

pub const ExprResult = union(enum) {
    integer: i64,
    float: f64,
    string: []const u8,
    boolean: bool,
};
```

Supported operators (precedence high to low):
- Unary: `!`, `-`, `+`
- Multiplicative: `*`, `/`, `%`
- Additive: `+`, `-`
- Comparison: `<`, `>`, `<=`, `>=`
- Equality: `==`, `!=`, `eq`, `ne` (eq/ne are STRING comparison -- CRITICAL for sky130, see RESEARCH Pitfall 6)
- Logical AND: `&&`
- Logical OR: `||`
- Ternary: `? :`

Operands: integer literals, float literals, `$var` references, `"string"` literals, `{braced}` literals, `[cmd]` substitution.

`eq` compares two operands as strings, returns 1 if equal, 0 otherwise.
`ne` compares two operands as strings, returns 1 if not equal, 0 otherwise.
`{}` (empty braces) is the empty string -- `$var ne {}` tests if var is non-empty.

NOTE: The lookup_var and lookup_env function pointers will be supplied by the evaluator (Task 2). For now, define the interface but the actual variable resolution happens in the evaluator.

Create `plugins/EasyImport/src/TCL/commands.zig` (~250 lines). Built-in command implementations:

```zig
pub fn execFile(args: []const []const u8, allocator: Allocator) ![]const u8;
pub fn execInfo(args: []const []const u8, var_exists: *const fn([]const u8) bool, env_exists: *const fn([]const u8) bool) ![]const u8;
pub fn execString(args: []const []const u8) ![]const u8;
```

`file` subcommands (required for sky130 -- see RESEARCH):
- `file dirname path` -> returns directory portion of path (`std.fs.path.dirname`)
- `file normalize path` -> returns normalized absolute path (`std.fs.path.resolve` with cwd)
- `file join a b ...` -> joins path segments (`std.fs.path.join`)
- `file isdir path` -> returns "1" if directory exists, "0" otherwise (`std.fs.cwd().access`)
- `file isfile path` -> returns "1" if file exists, "0" otherwise
- `file tail path` -> returns filename portion (`std.fs.path.basename`)
- `file extension path` -> returns file extension (`std.fs.path.extension`)

`info` subcommands:
- `info exists varname` -> returns "1" if variable exists in scope, "0" otherwise. Does NOT read the value (see RESEARCH Pitfall 5).
- `info exists env(NAME)` -> returns "1" if environment variable NAME is set, "0" otherwise. Must parse the `env(NAME)` syntax to extract NAME, then check `std.posix.getenv`.
- `info script` -> returns the path of the currently-executing script (stored in evaluator state)

`string` subcommands (minimal set):
- `string equal a b` -> "1" if equal, "0" otherwise
- `string tolower s` -> lowercased string
- `string length s` -> length as string

`puts` -> no-op (safe to ignore, it's just logging)

Create `plugins/EasyImport/src/TCL/evaluator.zig` (~380 lines). The core evaluator:

```zig
pub const Evaluator = struct {
    arena: std.heap.ArenaAllocator,
    variables: std.StringHashMapUnmanaged([]const u8),
    script_path: ?[]const u8,  // for [info script]
    source_visited: std.StringHashMapUnmanaged(void),  // loop guard for source
    backing: std.mem.Allocator,

    pub fn init(backing: std.mem.Allocator) Evaluator;
    pub fn deinit(self: *Evaluator) void;

    /// Set the path of the script being evaluated (for [info script])
    pub fn setScriptPath(self: *Evaluator, path: []const u8) void;

    /// Evaluate a complete Tcl script (multiple commands)
    pub fn evalScript(self: *Evaluator, script: []const u8) ![]const u8;

    /// Get a variable value
    pub fn getVar(self: *const Evaluator, name: []const u8) ?[]const u8;

    /// Set a variable value
    pub fn setVar(self: *Evaluator, name: []const u8, value: []const u8) !void;
};
```

Command dispatch using `std.StaticStringMap` (comptime lookup table per RESEARCH "Don't Hand-Roll"):

const command_map = std.StaticStringMap(CommandFn).initComptime(.{

Key command implementations:

`set name ?value?` -> if value provided, set variable and return it. If no value, return current value. Use `self.variables.put`.

`append name value` -> append value to existing variable (or create). Critical for `append XSCHEM_LIBRARY_PATH :$dir` pattern.

`lappend name value ...` -> append values as list elements (space-separated).

`if {cond} {body} ?elseif {cond} {body}? ?else {body}?` -> evaluate condition through expr parser. If true, evaluate body. Handle multi-line bodies (brace-delimited). Support chained elseif/else.

`source filepath` -> CRITICAL (see RESEARCH Pitfall 4):
1. Resolve `$VAR` in filepath
2. Check source_visited set for loop prevention
3. Read file contents (`std.fs.cwd().readFileAlloc`)
4. If file not found, emit diagnostic and continue (do NOT crash -- see RESEARCH Open Question 3)
5. Add to source_visited
6. Call `self.evalScript(file_contents)` recursively
7. File is evaluated in current scope (same variable table)

Variable substitution in command arguments:
- `$name` -> look up in variables table
- `${name}` -> same, braced form
- `$env(NAME)` -> `std.posix.getenv(NAME)`, return error.UndefinedVariable if not set (but `[info exists env(NAME)]` just returns "0")
- `[cmd args]` -> evaluate nested command and substitute result

Unsupported construct handling per D-04:
- `proc`, `switch`, `array`, `regexp`, `for`, `foreach`, `while`, `catch`, `global`, `namespace`, `package`, `eval`, `uplevel`, `upvar` -> return a specific error like `error.UnsupportedConstruct` AND before returning, print/log a diagnostic: "Unsupported Tcl construct '{name}' at line {N}". Use `std.debug.print` for the diagnostic.
