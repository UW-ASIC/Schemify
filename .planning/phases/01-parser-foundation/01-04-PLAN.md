The `parseRc` function:
```zig
/// Parse an xschemrc file using full Tcl evaluation.
/// `xschemrc_dir` is the directory containing the xschemrc file.
/// `xschemrc_path` is the absolute path to the xschemrc file itself.
pub fn parseRc(backing: Allocator, bytes: []const u8, xschemrc_dir: []const u8, xschemrc_path: []const u8) !RcResult { ... }
```

Implementation steps:

1. **Create Tcl evaluator and set up context:**
   - `var tcl = Tcl.init(backing);`
   - Set `info script` path: `tcl.setScriptPath(xschemrc_path)`
   - Pre-seed standard variables:
     - `XSCHEM_SHAREDIR` = detect xschem share dir (probe `/usr/share/xschem` or `$XSCHEM_SHAREDIR` env var)
     - `USER_CONF_DIR` = `$HOME/.xschem` (or `$USER_CONF_DIR` env var if set)
     - `PDK_ROOT` = `$PDK_ROOT` env var if set

2. **Evaluate the entire xschemrc through the Tcl evaluator:**
   - `tcl.eval(bytes)` -- this handles ALL the Tcl constructs including if/else blocks, source commands, variable substitution
   - On error from unsupported constructs (proc, switch, etc.), catch the error and continue -- these are DRC helpers that don't affect path resolution

3. **Read resolved variables from Tcl variable table:**
   - `XSCHEM_LIBRARY_PATH` -> split by `:` separator -> each segment is a library path
   - `XSCHEM_START_WINDOW` -> start_window
   - `XSCHEM_NETLIST_DIR` -> netlist_dir
   - `PDK_ROOT` -> pdk_root (might have been set/modified by the script)

4. **Resolve library paths:**
   - Split `XSCHEM_LIBRARY_PATH` by `:` (colon-separated in Tcl/XSchem convention)
   - Filter empty segments
   - Resolve relative paths against xschemrc_dir
   - Arena-dupe each resolved path
   - Store as `lib_paths: []const []const u8`

5. **Handle source command failures gracefully:**
   - The Tcl evaluator's `source` implementation should continue on file-not-found (see Plan 03)
   - If `source $PDK_ROOT/sky130A/libs.tech/xschem/xschemrc` fails because PDK is not installed, the lib_paths will just be what was resolved before the source command
   - This is acceptable: the sky130 xschemrc fixture (xschem_sky130/xschemrc) is self-contained and does NOT use `source`
