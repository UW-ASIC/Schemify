Create `plugins/EasyImport/src/XSchem/props.zig` (~200 lines) implementing a property string tokenizer. Consult old XSchem.zig lines 231-306 (PropTokenizer) as behavioral reference per D-02 but rewrite from scratch per D-01.

The PropertyTokenizer must handle these escaping rules (from RESEARCH property escaping reference):
1. Brace escaping: `\{` and `\}` within values -> literal `{` and `}`
2. Quote escaping: `\"` within quoted values -> literal `"`
3. Backslash: `\\` -> literal `\`
4. Multi-line: `key="value\n...continues..."` with closing `"` on later line
5. Single-quoted: `key='value'` (no escape processing inside)
6. Bare values: `key=value` (terminated by space)
7. Name special: `name=X1` is the instance designator
8. Empty braces: `{}` means no properties

Structure:
```zig
pub const PropertyTokenizer = struct {
    source: []const u8,
    pos: usize,

    pub fn init(source: []const u8) PropertyTokenizer { ... }
    pub fn next(self: *PropertyTokenizer) ?struct { key: []const u8, value: []const u8 } { ... }
};

/// Parse a property string into Prop array. All strings are arena-duped.
pub fn parseProps(arena: std.mem.Allocator, source: []const u8) !struct { props: []const Prop, count: u16 } { ... }
```

The `parseProps` function:
- Creates a PropertyTokenizer over the source
- Iterates all key=value pairs
- Allocates each key and value into the arena (arena.dupe)
- Returns the slice and count
- If source is `{}` or empty, returns empty slice with count=0
- Per D-06: return `error.UnbalancedBraces` if a quoted value is never closed

Import `Prop` from `types.zig` via `const types = @import("types.zig"); const Prop = types.Prop;`.
