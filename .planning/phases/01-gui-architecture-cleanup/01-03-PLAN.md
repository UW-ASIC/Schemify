**Part B: Migrate module-level vars to GuiState sub-structs** (per D-09, D-11)

CRITICAL: WinRect in state/types.zig is NOT dvui.Rect. Each dialog that uses `dvui.Rect` for win_rect must convert. The conversion is straightforward because both have the same fields {x, y, w, h} as f32. Use `dvui.Rect{ .x = wr.x, .y = wr.y, .w = wr.w, .h = wr.h }` where `wr` is the WinRect from state. Or use `@bitCast` if layout is identical. To be safe, use explicit field copy.

Create a helper function pattern in each file (or inline the conversion):
```zig
fn winRectToDvui(wr: *st.WinRect) *dvui.Rect {
    return @ptrCast(wr);
}
```
This works because WinRect and dvui.Rect have identical layout (4 f32 fields in same order). Use `@ptrCast` for zero-cost conversion. Verify by checking dvui.Rect definition -- if it has additional fields or different field order, use explicit field copy instead.
