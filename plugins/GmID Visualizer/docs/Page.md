# Analog circuit block parameter reference — with Gm/Id methodology mapping

> **How to read this document**
>
> Every block section lists parameters in three columns:
>
> - **Parameter / Symbol / Equation** — the spec and its governing equation
> - **Description** — what it means physically
> - **Gm/Id design handle** — how to translate this spec into a Gm/Id sizing decision, which transistor(s) it constrains, and how to allocate current (Id) and inversion level (Gm/Id) across the signal path
>
> The Gm/Id methodology states that every MOSFET (or BJT) is fully described by three variables: **Gm/Id** (inversion level / efficiency), **Id** (drain current), and **L** (channel length). All other device parameters — Gm, Gds, Cgs, Cgd, Cdb, ft, noise — follow from lookup tables or analytical fits indexed by (Gm/Id, L). The design flow is:
>
> 1. Derive required Id allocation from power and topology constraints
> 2. Derive required Gm from gain/bandwidth/noise targets
> 3. Compute Gm/Id = Gm / Id → maps to inversion level (weak/moderate/strong)
> 4. Read W from Id / (Id/W) lookup at that (Gm/Id, L) point
> 5. Verify Cgs, ft, Vdsat against signal path and headroom budgets

## Universal parameters — apply to every powered analog block

These parameters exist regardless of topology. The Gm/Id column explains how each constrains the global current/transistor budget.

| Parameter                   | Symbol   | Equation                   | Description                                                         | Gm/Id design handle                                                                                                                                                                                                                                                                                                                                                                  |
| --------------------------- | -------- | -------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Supply voltage              | VDD, VSS | VDD − VSS = supply span    | Operating range; sets headroom for all internal nodes               | Fixes the maximum VDS available to each device. Low VDD compresses Vdsat headroom, pushing transistors toward weak inversion (high Gm/Id) to preserve gain at reduced Id. Headroom allocation: each cascode stack consumes Vdsat + Vmargin per device, so VDD − VSS = Σ(Vdsat_i) + Vswing sets the number of series devices you can stack.                                           |
| Supply current              | IDD      | P = VDD · IDD              | Quiescent + dynamic draw; directly sets power budget                | **Primary current budget knob.** IDD is the total Id to be distributed across all branches. For a telescopic OTA: IDD = Id_diff_pair = Id_load (single branch). For a folded cascode: IDD = Id_input_pair + Id_fold_current, split by a ratio α. For each transistor, Id_i is directly the x-axis of Gm/Id sizing curves — you choose Id_i, then pick Gm/Id_i to set Gm_i.           |
| Power dissipation           | P        | P = VDD · IDD              | Total power; determines thermal budget                              | Sets the ceiling on ΣId across all branches. Once P is fixed, it flows into current allocation decisions: how much Id goes to the input pair (which sets Gm and noise) vs cascode devices (which set output impedance and bandwidth) vs tail source. Efficiency-conscious design maximises Gm per unit Id → operate input pair in moderate inversion (Gm/Id ≈ 10–20 S/A for CMOS).   |
| Operating temperature range | TA, TJ   | TJ = TA + P · θJA          | Junction temperature under load; affects every other parameter      | All Gm/Id lookup tables are temperature-dependent. Gm/Id at fixed bias point shifts because µ(T) and Vt(T) change. PTAT/CTAT biasing can be used to stabilise Gm (and therefore gain/BW) over temperature. At the sizing stage, corner simulations must sweep T alongside VDD and process corner.                                                                                    |
| Noise floor                 | Sn(f)    | Vn = √(4kT · R · Δf)       | Thermal + flicker noise at input referred; sets dynamic range floor | Input-referred noise spectral density of a MOSFET in saturation is Sv = 4kT · γ / Gm (thermal) + Kf / (Cox · W · L · f) (flicker). Increasing Gm (by increasing Id or Gm/Id) directly reduces thermal noise. For a given noise target: Gm_min = 4kT · γ / Sv_target. Then Id = Gm / (Gm/Id). Flicker noise requires large W · L (or choosing high-Gm/Id to get large W at modest L). |
| Input-referred noise        | en, in   | SNR = Vsig / (en · √BW)    | Voltage and current noise density (nV/√Hz, pA/√Hz)                  | en² = 4kTγ/Gm for thermal; Gm of the input transistor is the dominant handle. To halve en, quadruple Gm (either double Id or double Gm/Id). in is negligible for MOSFET gate inputs but significant for BJT base. In a diff pair, the input pair Gm/Id and Id are chosen to hit the en target first; all other constraints follow.                                                   |
| Dynamic range               | DR       | DR = 20·log(Vmax / Vnoise) | Ratio of largest to smallest usable signal                          | DR = 20·log(Vswing / en·√BW). Vswing is set by headroom (VDD, topology, Vdsat allocation). en is set by input transistor Gm. Both ends of DR are therefore Gm/Id-linked: high Gm/Id → lower Vdsat → more Vswing AND lower en → DR improves doubly. This is why weak/moderate inversion is attractive for low-power, high-DR circuits.                                                |
| Process corner              | SS/TT/FF | —                          | PVT corner: slow-slow to fast-fast                                  | Gm/Id curves shift across corners. In SS corners, µ is low → for the same Id and L, Gm/Id is lower (less efficient). Design must be verified across all corners. Constant-Gm biasing (PTAT Vgs − Vt reference) can partially compensate. Vt mismatch (AVT parameter) sets offset floor.                                                                                              |
| ESD rating                  | HBM, CDM | —                          | Electrostatic discharge withstand on every pin                      | ESD structures add pad capacitance Cpad (typically 100 fF–2 pF). This capacitance is part of the input loading and directly degrades bandwidth. At the Gm/Id level: ft must be high enough that the input transistor's Cgs is not the bottleneck — the ESD cap often dominates. Account for Cpad in noise analysis (thermal noise of ESD resistance).                                |
| Latch-up immunity           | Itrigger | —                          | Parasitic PNPN trigger current; CMOS process concern                | Not directly a Gm/Id parameter, but transistor sizing and layout (guard rings, well taps) interact with W sizing decisions. Very wide transistors at output stages increase substrate injection risk.                                                                                                                                                                                |

## 1. Single in → Single out

_Amplifier, buffer, filter, single-ended driver_

### 1.1 Gain and transfer

| Parameter              | Symbol | Equation              | Description                                                      | Gm/Id design handle                                                                                                                                                                                                                                                                                                                                                                             |
| ---------------------- | ------ | --------------------- | ---------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Voltage gain           | Av     | Av = Vout / Vin       | Linear gain (V/V) or dB; the primary transfer parameter          | For a single-stage OTA: Av = Gm · Rout = Gm · (gds_n + gds_p)⁻¹. Gm is set by Id and Gm/Id of the input transistor. Rout is set by λ (channel-length modulation) of output devices — longer L → smaller λ → higher Rout. Tradeoff: longer L reduces ft and hence bandwidth. For cascoded stages: Av = Gm · (gm2·rds2·rds4) → very high gain without extra current, at the cost of output swing. |
| Gain bandwidth product | GBW    | GBW = Av_DC · f−3dB   | Constant for dominant-pole amplifiers; trades gain for bandwidth | GBW = Gm / (2π · CL) for a single-pole OTA. This is the central Gm/Id sizing equation: **Gm = 2π · GBW · CL**. Once CL and GBW are specified, Gm is fixed. Then Id = Gm / (Gm/Id), which directly sets power. Choosing high Gm/Id (weak inversion) minimises Id for a given Gm — the core power-efficiency argument.                                                                            |
| −3 dB bandwidth        | f−3dB  | f−3dB = GBW / Av      | Frequency where gain drops 3 dB; sets usable signal bandwidth    | Non-dominant poles are set by internal node capacitances: fp2 = Gm2 / (2π · Cint). To push fp2 high, need large Gm2 or small Cint. Increasing Gm2 costs current; reducing Cint means shorter L and smaller W (lower parasitic). At the Gm/Id level: for each internal transistor, compute ft = Gm / (2π · Cgs) — this must exceed the required internal pole frequency.                         |
| Phase margin           | PM     | PM = 180° + ∠H(jω) at | H                                                                | =0 dB                                                                                                                                                                                                                                                                                                                                                                                           | Stability margin; <45° risks oscillation in closed-loop | The non-dominant pole fp2 = gm5/(2π·Cgd1) or gm_output/(2π·CL_internal). Miller compensation: Cc sets the dominant pole; gm of the output stage must satisfy gm_out ≥ 2π · GBW · CL to keep PM > 60°. In Gm/Id terms: size the output stage transistor so its Gm meets this bound, then choose Gm/Id to minimise Id — but at high Gm/Id (weak inversion), ft is low, potentially creating additional parasitic poles. |
| Gain margin            | GM     | GM = −                | H(jω)                                                            | at ∠H = −180°                                                                                                                                                                                                                                                                                                                                                                                   | Gain headroom before instability                        | Determined by the location of the 180° phase crossing — additional poles/zeros from layout parasitics. In Gm/Id design, parasitic capacitances (Cgd, Cdb) are read from lookup tables at the chosen (Gm/Id, L) operating point, and their contribution to gain/phase is verified before finalising W.                                                                                                                 |
| Group delay            | τg(f)  | τg = −dφ/dω           | Frequency-dependent delay; flat = linear phase                   | Non-flat group delay implies poles/zeros spread in frequency. In a filter designed with Gm/Id, each Gm cell's Gm/Id and Id are chosen so that the pole frequencies hit the filter prototype values exactly. Sensitivity of pole frequency to Gm/Id variation (mismatch, corner) determines group delay flatness robustness.                                                                     |

### 1.2 Linearity and distortion

| Parameter        | Symbol   | Equation                         | Description                                                   | Gm/Id design handle                                                                                                                                                                                                                                                                                                                                                                                         |
| ---------------- | -------- | -------------------------------- | ------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| THD              | THD      | THD = √(V2²+V3²+…) / V1          | Total harmonic distortion                                     | Distortion in a MOSFET amplifier is dominated by Vgs nonlinearity. In strong inversion: HD3 ≈ (Vgs−Vt)⁻² · (Vsig/4)². In weak inversion (high Gm/Id): the transconductance is more exponential (BJT-like) → higher HD2 but potentially lower HD3. For a given Vsig, operating in moderate inversion balances linearity and efficiency. Increasing Id (lower Gm/Id) reduces distortion by increasing Vgs−Vt. |
| HD2, HD3         | HD2, HD3 | HD2 = V2/V1, HD3 = V3/V1         | Individual harmonic terms                                     | HD2 is an asymmetry measure — a perfectly balanced diff pair cancels HD2 to first order. HD3 is the dominant term in strong inversion: HD3 ≈ (1/48) · (Vsig/(Vgs−Vt))². This directly links to Gm/Id: Vgs−Vt ≈ 2/(Gm/Id) in strong inversion, so HD3 ≈ (Gm/Id)² · Vsig² / 192. High Gm/Id → higher HD3 for same signal swing.                                                                               |
| IP3 (output)     | OIP3     | OIP3 = Pout + IIP3 − Pin         | Third-order intercept; characterises two-tone intermodulation | IIP3 = √(24 · Id / (Gm · g3)) where g3 is the third derivative of Id vs Vgs. In Gm/Id design: IIP3 is a direct function of inversion level. Strong inversion → lower g3/Gm → better IIP3. Weak inversion → exponential ID(Vgs) → poor IIP3. This creates a fundamental tradeoff: high Gm/Id improves noise efficiency but degrades linearity.                                                               |
| 1-dB compression | P1dB     | Pout = Pin + Gain − 1 dB         | Input power where gain compresses by 1 dB                     | P1dB ≈ IIP3 − 10 dB. Same Gm/Id tradeoff as IIP3. For a given P1dB requirement, minimum Id is set by the required inversion level to achieve that IIP3. In cascaded stages, the last (highest signal level) stage dominates compression — size it in stronger inversion (lower Gm/Id) even at higher power cost.                                                                                            |
| SFDR             | SFDR     | SFDR = Pfund − Pworst_spur (dBc) | Spurious-free dynamic range                                   | SFDR = (2/3) · (IIP3 − Noise Floor). Both IIP3 and noise floor are Gm/Id-dependent as above. Optimal Gm/Id for maximum SFDR is typically moderate inversion — the sweet spot between noise efficiency (high Gm/Id) and linearity (low Gm/Id). This is a classic Gm/Id optimisation problem solved graphically by plotting SFDR vs Gm/Id at fixed power.                                                     |

### 1.3 Impedance

| Parameter         | Symbol | Equation                  | Description                                             | Gm/Id design handle                                                                                                                                                                                                                                                                                                              |
| ----------------- | ------ | ------------------------- | ------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Input impedance   | Zin    | Zin = Vin / Iin           | Must be >> source impedance to avoid loading; Rin ∥ Cin | For a MOSFET gate: Rin ≈ ∞ DC, Cin = Cgs + Cgd(1+Av). Cgs = (2/3)·Cox·W·L (saturation). At a chosen (Gm/Id, L, Id): W = (Id/W)                                                                                                                                                                                                   | lookup · Id⁻¹, so Cgs is determined. To minimise Cin (for high-Z input or high-frequency pole): choose shorter L or lower Gm/Id (strong inversion → smaller W for same Id). |
| Output impedance  | Zout   | Zout = ΔVout / ΔIout      | Must be << load impedance; drives signal integrity      | Zout = rds = 1/(λ · Id) for a simple CS stage. λ ∝ 1/L, so longer L reduces λ and increases Zout. In Gm/Id design: Zout is a function of (Gm/Id, Id, L) and is read from intrinsic gain lookup: Av0 = Gm · rds = (Gm/Id) · (1/λ). For high Zout without cascode, choose long L and read rds from the gds/Id vs (Gm/Id, L) table. |
| Input capacitance | Cin    | fpole = 1/(2π · Rs · Cin) | Forms pole with source resistance; limits HF response   | Cin = Cgs(1) + Cgd(1)(1+Av). Cgs is read from (Gm/Id, L) lookup as Cgs/Id. Since Cgs/Id decreases with increasing Id (short channel effects), there is an optimal Id that minimises the bandwidth-limiting pole while meeting noise target. This is a classic Gm/Id co-optimisation.                                             |

### 1.4 Offset and DC errors

| Parameter            | Symbol  | Equation                | Description                                | Gm/Id design handle                                                                                                                                                                                                                                                                                                                                                                       |
| -------------------- | ------- | ----------------------- | ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Input offset voltage | Vos     | Vout_error = Vos · Av   | DC mismatch referred to input              | Vos is caused by threshold mismatch ΔVt and current factor mismatch Δβ/β. From Pelgrom's model: σ(ΔVt) = AVT / √(W·L). For a given Vos budget: W·L ≥ (AVT / σ_Vos)². At the Gm/Id level: larger W·L (more area) is needed for better matching, but this also increases Cgs and Cgd. The tradeoff is explicit: mismatch → large W·L; bandwidth → small W·L. Gm/Id sets W given Id and L.   |
| Offset drift         | dVos/dT | ΔVos = (dVos/dT) · ΔT   | µV/°C; how offset changes with temperature | dVos/dT ≈ ΔVt · (1/T) − temperature coefficient of µ · mismatch terms. Biasing in moderate inversion with PTAT Vgs−Vt reference partially compensates drift. The Gm/Id operating point shifts with T; circuits designed at TT must be verified at hot/cold corners with updated Gm/Id tables.                                                                                             |
| PSRR                 | PSRR    | PSRR = ΔVos / ΔVDD (dB) | Power-supply ripple rejection              | PSRR is limited by Cgd of the input transistor coupling supply to output, and by finite tail current source impedance. In Gm/Id design: Cgd is read from the lookup at the operating point. Cascoding the tail source (longer L → higher rds_tail) improves PSRR. The tail transistor's Gm/Id and L are chosen to achieve required output impedance (rds_tail · gm_cascode = ro product). |

### 1.5 Transient and slew

| Parameter     | Symbol | Equation                         | Description                                     | Gm/Id design handle                                                                                                                                                                                                                                                                                                                                                                  |
| ------------- | ------ | -------------------------------- | ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Slew rate     | SR     | SR = dVout/dt                    | max (V/µs)                                      | Maximum output rate of change; limits large-signal speed                                                                                                                                                                                                                                                                                                                             | SR = Id_tail / CL for a single-stage OTA. This is a direct current budget constraint: **Id_tail = SR · CL**. Once CL and SR are specified, Id_tail is fixed before any Gm/Id decision. The input pair Gm/Id is then chosen to meet GBW given this fixed Id_tail (since Id_pair = Id_tail/2 in a diff pair). If SR requires a larger Id than noise or GBW dictates, power is slew-limited. |
| Settling time | ts     | ts to N-bit accuracy             | Time to settle within error band after a step   | ts is dominated by the slowest of: slew phase (set by SR), linear settling phase (set by GBW and PM), and noise floor. All three are linked to Gm/Id: GBW → Gm → Id/Gm/Id; SR → Id; noise → Gm. A co-optimisation sweeps Gm/Id for the input transistor while holding ts constant, minimising total Id.                                                                              |
| Overshoot     | %OS    | %OS = (Vpeak−Vfinal)/Vstep · 100 | Ringing overshoot on step; tied to phase margin | Overshoot is set by PM. PM depends on the ratio of non-dominant pole to GBW: fp2/GBW. fp2 is set by the Gm and parasitic capacitance of internal nodes — both read from Gm/Id tables. Increasing Id of the output stage transistor increases its Gm and pushes fp2 higher, improving PM. This is a direct Gm/Id current allocation trade: pay more Id in the output stage to buy PM. |

## 2. Differential in → Single out

_Diff-pair, instrumentation amp input stage, standard op-amp topology_

All parameters from Section 1 apply to the single-ended signal path. The following are additional or modified.

### 2.1 Differential gain and CMRR

| Parameter         | Symbol    | Equation                              | Description                                         | Gm/Id design handle                                                                                                                                                                                                                                                                                                                                                                                                              |
| ----------------- | --------- | ------------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Differential gain | Ad        | Ad = Vout / (Vin+ − Vin−)             | Gain on the wanted differential signal              | Ad = Gm_pair · Rout. Gm_pair is the transconductance of the differential pair transistors — directly set by Id_pair and Gm/Id_pair. This is the first Gm/Id decision: given the GBW requirement, Gm_pair = 2π · GBW · CL, then Id_pair = Gm_pair / (Gm/Id_pair). Choosing high Gm/Id_pair (weak inversion) minimises Id_pair for a given Gm.                                                                                     |
| Common-mode gain  | Acm       | Acm = Vout / Vcm                      | Gain on the unwanted common-mode signal             | Acm = 1 / (2 · gm_pair · Rss) where Rss is the impedance of the tail current source. Rss = rds_tail · (1 + gm_cascode · rds_cascode) for a cascoded tail. In Gm/Id terms: the tail transistor's rds is 1/(gds) = (Gm/Id_tail)/(Gm_tail · gds/Gm) read from lookup. Longer L for tail transistor → higher rds → better CMRR. This is an independent Gm/Id sizing decision for the tail device.                                    |
| CMRR              | CMRR      | CMRR = 20·log(Ad/Acm) (dB)            | The key spec for diff input blocks                  | CMRR = Gm_pair · Rss · (load_mismatch_factor)⁻¹. The dominant limit at low frequency is load mismatch (Δgds of PMOS loads); at high frequency it is capacitive mismatch (ΔCgd of input transistors). In Gm/Id sizing: larger W·L for input pair improves capacitive matching (Cgd mismatch ∝ 1/√(WL)); larger L for tail improves Rss. Both moves have opposing effects on speed and area — explicit tradeoff in the Gm/Id flow. |
| CMRR vs frequency | CMRR(f)   | CMRR drops ~20 dB/dec above fp        | CMRR degrades at high frequency due to Cin mismatch | The HF CMRR corner is at fp = 1/(2π · ΔCgd · Rs). ΔCgd is a mismatch parameter scaling as Cgd/√(WL). In Gm/Id design: the input transistor's Cgd is read from the lookup at (Gm/Id, L). Larger W at the same Gm/Id (i.e., higher Id or longer L) reduces relative mismatch → pushes the CMRR corner frequency higher.                                                                                                            |
| Common-mode range | VCM range | VSS + headroom ≤ VCM ≤ VDD − headroom | Input voltage range over which diff function holds  | VCM range sets the minimum Vdsat allocations in the input pair and tail source: Vcm_min = VSS + Vdsat_tail + Vgs_input. Low Gm/Id (strong inversion) → high Vdsat → compressed Vcm range. High Gm/Id (weak inversion) → low Vdsat → wider Vcm range. The tail transistor Gm/Id is explicitly chosen to hit Vcm_min while meeting Rss (CMRR) requirement — solved via Gm/Id lookup of Vdsat at operating point.                   |

### 2.2 Input mismatch

| Parameter            | Symbol   | Equation                 | Description                                                       | Gm/Id design handle                                                                                                                                                                                                                                                                                                                   |
| -------------------- | -------- | ------------------------ | ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Input offset voltage | Vos_diff | Vout_err = Vos · Ad      | Residual DC due to transistor mismatch in diff pair               | σ(Vos) = √(σ²(ΔVt) + (Id·σ(Δβ/β)/(Gm))²) = AVT/√(WL) at low Id. In Gm/Id sizing: for a required Vos budget, minimum WL = (AVT/σ_Vos)². Then W/L = (W from Gm/Id sizing) / L. If WL_mismatch > WL_noise, mismatch is the binding constraint on transistor area.                                                                        |
| Input bias current   | IB+, IB− | Vos_err = (IB+−IB−) · Rs | Base/gate currents; mismatch creates offset when source Z differs | MOSFET: IB = Ig ≈ 0 (gate leakage dominated by tunnelling in thin-oxide processes). For short L (high-κ gate), Ig can reach fA–pA — read from model. BJT: IB = IC/β, directly proportional to Id. In BJT Gm/Id design: higher Gm/Id (lower Vbe overdrive) means lower IC for same Gm, hence lower IB — beneficial for high-Z sources. |
| Input offset current | Ios      | Ios = IB+ − IB−          | Mismatch of the two bias currents                                 | For MOSFET, Ios is negligible. For BJT: Ios = IC · Δβ/β² — size base area for β matching. In BJT Gm/Id: Ios is minimised by operating at low Ic (high Gm/Id = high β regime) and by using emitter degeneration which reduces effective Gm but improves matching.                                                                      |

### 2.3 Single-ended output

| Parameter             | Symbol     | Equation             | Description                                           | Gm/Id design handle                                                                                                                                                                                                                                                                                                                                             |
| --------------------- | ---------- | -------------------- | ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Output swing          | Vout_swing | Vout_min to Vout_max | Rail-to-rail or headroom-limited output range         | Vout_swing = VDD − Vdsat_p − Vdsat_n (for cascoded output). ## 3. Single in → Differential out

_Single-to-differential converter, balun driver, FDA driven single-ended_

All parameters from Section 1 apply to the input. The following govern the differential output port.

### 3.1 Conversion quality

| Parameter            | Symbol | Equation                        | Description                                           | Gm/Id design handle                                                                                                                                                                                                                                                                                                             |
| -------------------- | ------ | ------------------------------- | ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Gain (single→diff)   | Ad     | Ad = (Vout+−Vout−) / Vin        | Differential output swing per unit single-ended input | Ad = Gm · Zout_diff. Gm is the input transistor's transconductance (set by Id and Gm/Id as before). Zout_diff = 2 · Rout_half_circuit. The two output branches must be balanced: the NMOS and PMOS devices on each side must be Gm/Id co-designed to present equal impedances, otherwise the signal is not purely differential. |
| Output balance error | ΔA     | ΔA =                            | A+ − A−                                               | / ((A++A−)/2)                                                                                                                                                                                                                                                                                                                   | Mismatch between the two output paths | Balance error arises from Gm mismatch between the + and − output transistors: ΔA/A ≈ ΔGm/Gm = Δ(Gm/Id)/Gm/Id + ΔId/Id. Both terms are driven by transistor mismatch. Pelgrom scaling applies: larger W·L → smaller ΔGm/Gm. At the Gm/Id level: balance error sets the minimum WL product for the output devices. |
| Output common-mode   | Vocm   | Vocm = (Vout+ + Vout−)/2        | DC level of the differential output pair              | Vocm is set by the CMFB loop reference. The CMFB amplifier itself must be sized with Gm/Id: its Gm must be high enough to provide sufficient loop gain to suppress Vocm error, but its power must be budgeted. CMFB current is typically 10–20% of main signal path current — allocated separately in the Id budget.            |
| Vocm accuracy        | ΔVocm  | ΔVocm = Vocm_actual − Vocm_set  | How accurately the output CM tracks its reference     | ΔVocm = Vos_CMFB / (CMFB loop gain). Loop gain = Gm_CMFB · Rout_CMFB. To reduce ΔVocm: increase Gm_CMFB (more Id or higher Gm/Id) or increase Rout_CMFB (longer L). This is a separate Gm/Id sizing sub-problem for the CMFB amplifier.                                                                                         |
| Phase mismatch       | Δφ     | Δφ = φ(Vout+) − φ(Vout−) − 180° | Deviation from perfect anti-phase                     | Phase mismatch at HF is caused by Cgs/Cgd mismatch between the two output transistors. Since Cgs ∝ W·L and Cgd ∝ W, larger devices reduce relative mismatch. In Gm/Id design: ΔCgs/Cgs ∝ 1/√(WL). For a phase mismatch target Δφ_max at frequency f: the required Cgs matching tolerance sets minimum WL.                       |
| Amplitude mismatch   | ΔGain  | ΔG = 20·log(A+/A−) (dBc)        | Gain mismatch between + and − output branches         | Same Pelgrom analysis as balance error above. ΔGm/Gm ≈ AVβ/√(WL) for current factor mismatch. Gm/Id operating point affects which mismatch term dominates: at high Gm/Id (weak inversion) the ΔVt term dominates; at low Gm/Id (strong inversion) the Δβ term is relatively larger.                                             |

### 3.2 Differential output port

| Parameter                 | Symbol | Equation                        | Description                                           | Gm/Id design handle                                                                                                                                                                                                                                                                                                                  |
| ------------------------- | ------ | ------------------------------- | ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Differential output swing | Vod    | Vod = Vout+ − Vout− (peak-peak) | Total differential output voltage swing               | Vod_max = 2 · (VDD/2 − Vdsat_output). Each Vdsat is read from Gm/Id lookup. To maximise Vod, minimise Vdsat → use high Gm/Id (weak/moderate inversion) on output devices. Tradeoff: poor linearity and lower ft at weak inversion.                                                                                                   |
| Output impedance (diff)   | Zod    | Zod = (Vout+−Vout−) / Iod       | Differential output impedance seen by downstream load | Zod = 2 · rds_out (half-circuit). Increasing L of output transistors raises rds and hence Zod. In a cascode: Zod = 2 · gm_cascode · rds_cascode · rds_output, which can reach MΩ without extra current. The cascode transistor's own Gm/Id and L are chosen to maximise gm · rds product — typically moderate inversion with long L. |
| Output impedance (CM)     | Zocm   | Zocm = Vocm / Iocm              | Common-mode output impedance                          | Zocm is set by the CMFB loop: Zocm_closed ≈ Zocm_open / (1 + CMFB loop gain). A well-designed CMFB with high loop gain drives Zocm → 0. CMFB amplifier Gm sets the loop gain: Gm_CMFB · Rout_CMFB > 40 dB typically required.                                                                                                        |

Leakage inductance Lleak = (1−k²)·L1 creates a series resonance with the load capacitance — a zero in the passband. The receiving transistor's bandwidth must account for this: the pole-zero pair from Lleak and Cload must be outside the signal band, or it must be absorbed into the matching network design (which changes the Gm/Id sizing for the input stage to accommodate the modified Zin). |
| Balance (balun)      | Amplitude/Phase balance | ΔA (dB), Δφ (°)         | For balun use: quality of single-to-differential conversion | Imbalance from the passive balun directly appears as common-mode signal at the differential receiver input. The receiver's CMRR (Sections 2 and 4) must be sufficient to reject this imbalance. Specifically: CMRR_required ≥ 20·log(1/imbalance) + desired signal rejection. This sets the CMRR target, which through Section 2.1 sets the Gm/Id sizing of the receiver's tail current source and input pair matching.           |

### 12.5 Transmission line-specific

| Parameter                | Symbol | Equation             | Description                                        | Gm/Id design handle                                                                                                                                                                                                                                                                                                                       |
| ------------------------ | ------ | -------------------- | -------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Characteristic impedance | Z0     | Z0 = √(L'/C')        | Intrinsic impedance; 50 Ω or 75 Ω standard         | The driving transistor output impedance must equal Z0. For common-gate stage: Zout_in = 1/Gm → Gm = 1/Z0 = 1/50 = 20 mS. Then Id = Gm/(Gm/Id). For Z0 = 50 Ω: if Gm/Id = 20 S/A → Id = 1 mA; if Gm/Id = 10 S/A → Id = 2 mA. This is a direct power specification derived from Z0 and the Gm/Id operating point.                           |
| Time delay               | td     | td = l/vp = l√(L'C') | Propagation delay; critical for high-speed digital | The delay must be matched to within timing margins in clock distribution and high-speed differential data paths. The active repeater transistors (if any) must have td_driver < td_line_segment/(SNR_timing_margin). This is a Gm/Id bandwidth constraint: the repeater bandwidth = Gm/(2π·CL) must be sufficient to not add extra delay. |

## Appendix: Gm/Id co-optimisation summary

The following table summarises the primary Gm/Id design variables for each parameter category, and identifies the main tradeoffs that must be jointly resolved.

| Category           | Primary design variables                                           | Key tradeoff                                            |
| ------------------ | ------------------------------------------------------------------ | ------------------------------------------------------- |
| Gain / GBW         | Gm/Id and Id of input transistor; L of output transistors          | Power (Id) vs bandwidth (Gm), noise vs linearity        |
| Noise              | Gm/Id × Id of input transistor (maximise Gm); W·L for flicker      | Power vs noise figure; flicker corner vs area           |
| Offset / matching  | W·L of differential pair (Pelgrom)                                 | Area vs offset; mismatch vs speed (Cgs)                 |
| Output swing       | Vdsat = f(Gm/Id, Id) of all stacked devices                        | Headroom vs gain (Vdsat vs rds); swing vs linearity     |
| Stability (PM)     | Gm of output stage, parasitic C from Gm/Id tables                  | Output stage power vs PM; compensation cap vs BW        |
| Distortion / IIP3  | Gm/Id of signal path transistors (low → better)                    | Linearity vs power efficiency                           |
| Current allocation | Id split across branches (diff pair, cascode, tail, CMFB)          | Signal-path Gm vs reference quality vs output impedance |
| CMRR               | Rss (tail L and Gm/Id), Cgd mismatch (W·L of input pair)           | CMRR vs Vcm range vs area                               |
| Phase noise        | Gm of cross-coupled pair; W·L for flicker noise                    | Phase noise vs power; flicker corner vs area            |
| kT/C noise         | Chold (independent of transistor); switch Ron vs Qinj              | Noise floor vs speed vs pedestal                        |
| Efficiency (power) | Switch Gm/Id low (strong inv) for Rdson; control Gm/Id high for Iq | Conduction loss vs switching loss vs quiescent power    |

> **Note on BJTs:** For bipolar transistors in the Gm/Id framework, Gm/Ic = q/(kT) = 1/VT ≈ 38.5 S/A at 300 K — a fixed value in the active region, independent of current. This means BJTs always operate at the maximum possible transconductance efficiency. The design freedom shifts to Ic (current), L (emitter length for noise and matching), and VCE headroom. All equations in this document apply to BJTs by substituting Gm/Id → q/kT and recognising that base current IB = Ic/β adds a noise term in_b = √(2q·IB) absent in MOSFETs.
