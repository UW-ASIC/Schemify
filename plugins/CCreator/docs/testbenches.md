# Testbenches

CCreator provides a testbench framework for defining, simulating, and exporting SPICE testbenches. Testbenches wrap a circuit under test with sources, probes, and analysis directives.

## Defining a Testbench

```python
from ccreator import testbench, realistic, Port

@realistic.analog
class Amplifier:
    ports = [Port('inp', 'input', 'analog'), Port('out', 'output', 'analog'),
             Port('vdd', 'inout', 'voltage'), Port('vss', 'inout', 'voltage')]

    def build(self, n):
        n.MOSFET('M1', 'out', 'inp', 'vss', 'vss', 'sky130_fd_pr__nfet_01v8', W='10u', L='0.5u')
        n.MOSFET('M2', 'out', 'out', 'vdd', 'vdd', 'sky130_fd_pr__pfet_01v8', W='20u', L='0.5u')

@testbench
class AmpACTestbench:
    """AC analysis of amplifier gain and bandwidth."""

    def build(self, tb):
        # Instantiate DUT
        tb.instance(Amplifier(), 'dut', {
            'inp': 'inp', 'out': 'out', 'vdd': 'vdd', 'vss': 'gnd',
        })

        # Power supply
        tb.V('vdd', 'vdd', 'gnd', dc=1.8)

        # Input: DC bias + AC stimulus
        tb.V('vin', 'inp', 'gnd', dc=0.9, ac=1)

        # Probe output
        tb.probe('out')

    def analysis(self, tb):
        tb.ac(fstart=1, fstop=1e9, points=200)
```

## The `@testbench` Decorator

Wraps a class with `BaseTestbench`, which provides:

- `run()` — simulate and return a `SimulationResult`
- `export.spice(path)` — write the testbench as a SPICE netlist

The class must define:

| Method | Required | Purpose |
|--------|----------|---------|
| `build(self, tb)` | Yes | Add instances, sources, and probes |
| `analysis(self, tb)` | Yes | Define the analysis type (ac, tran, dc) |
| `assertions(self, result)` | No | Optional post-simulation validation |

## TestbenchBuilder API

The `tb` argument in `build()` and `analysis()` is a `TestbenchBuilder`:

### Instances

```python
tb.instance(circuit, name, connections)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `circuit` | `BaseCircuit` instance | The DUT or subcircuit |
| `name` | `str` | Instance name (e.g. `'dut'`, `'load'`) |
| `connections` | `dict[str, str]` | Port-to-net mapping: `{'inp': 'net_in', 'out': 'net_out'}` |

### Sources

```python
# DC voltage source
tb.V('vdd', 'vdd', 'gnd', dc=1.8)

# AC stimulus
tb.V('vin', 'inp', 'gnd', dc=0.9, ac=1)

# Sinusoidal source
tb.V('vsig', 'sig', 'gnd', sin={'offset': 0, 'amplitude': 0.1, 'frequency': 1e6})

# Pulse source
tb.V('vclk', 'clk', 'gnd', pulse={
    'initial': 0, 'pulsed': 1.8,
    'delay': 0, 'rise': 1e-9, 'fall': 1e-9,
    'width': 5e-9, 'period': 10e-9,
})

# Current source
tb.I('ibias', 'vdd', 'bias', dc=10e-6)
```

### Probes

```python
tb.probe('out')             # probe voltage at node 'out'
tb.probe('out', 'inp')      # probe multiple nodes
```

### Analysis

```python
# AC analysis
tb.ac(fstart=1, fstop=1e9, points=200, variation='dec')

# Transient analysis
tb.tran(step=1e-9, end=1e-6)

# DC sweep
tb.dc(source='Vvin', start=0, stop=1.8, step=0.01)
```

## Running a Testbench

```python
tb = AmpACTestbench()
result = tb.run()

# result is a SimulationResult
print(result.kind)       # 'ac'
print(result.x.shape)    # frequency array
print(result.y.keys())   # {'out'} — probed nodes

# Extract metrics
m = result.metrics('out')
print(f"f3dB: {m.f3db:.2e} Hz")

# Plot
result.plot()

# Text report
result.report()
```

### Assertions

Add post-simulation checks in the `assertions` method:

```python
@testbench
class AmpACTestbench:
    def build(self, tb): ...
    def analysis(self, tb): ...

    def assertions(self, result):
        m = result.metrics('out')
        assert m.f3db > 1e6, f"Bandwidth too low: {m.f3db:.2e}"

# Run with assertion checking
tb = AmpACTestbench()
result = tb.run(assert_on_complete=True)  # raises AssertionError if assertions fail
```

```python
tb = AmpACTestbench()

# Write SPICE netlist to file
tb.export.spice('output/tb_amp_ac.sp')
```

## Transient Testbench Example

```python
@testbench
class AmpTransientTestbench:
    parameters = {'freq': 1e6, 'amplitude': 50e-3}

    def build(self, tb):
        tb.instance(Amplifier(), 'dut', {
            'inp': 'inp', 'out': 'out', 'vdd': 'vdd', 'vss': 'gnd',
        })
        tb.V('vdd', 'vdd', 'gnd', dc=1.8)
        tb.V('vin', 'inp', 'gnd', dc=0.9, sin={
            'offset': 0.9,
            'amplitude': self.amplitude,
            'frequency': self.freq,
        })
        tb.probe('out', 'inp')

    def analysis(self, tb):
        period = 1.0 / self.freq
        tb.tran(step=period / 100, end=period * 10)

tb = AmpTransientTestbench(freq=1e6, amplitude=100e-3)
result = tb.run()
result.plot()
```

## DC Sweep Testbench Example

```python
@testbench
class AmpDCSweep:
    def build(self, tb):
        tb.instance(Amplifier(), 'dut', {
            'inp': 'inp', 'out': 'out', 'vdd': 'vdd', 'vss': 'gnd',
        })
        tb.V('vdd', 'vdd', 'gnd', dc=1.8)
        tb.V('vin', 'inp', 'gnd', dc=0.9)
        tb.probe('out')

    def analysis(self, tb):
        tb.dc(source='Vvin', start=0, stop=1.8, step=0.01)

result = AmpDCSweep().run()
# result.x = input voltage sweep
# result.y['out'] = output voltage
```

## Multi-DUT Testbenches

You can instantiate multiple circuits:

```python
@testbench
class ComparativeTestbench:
    def build(self, tb):
        tb.instance(AmplifierV1(), 'v1', {'inp': 'inp', 'out': 'out_v1', 'vdd': 'vdd', 'vss': 'gnd'})
        tb.instance(AmplifierV2(), 'v2', {'inp': 'inp', 'out': 'out_v2', 'vdd': 'vdd', 'vss': 'gnd'})
        tb.V('vdd', 'vdd', 'gnd', dc=1.8)
        tb.V('vin', 'inp', 'gnd', dc=0.9, ac=1)
        tb.probe('out_v1', 'out_v2')

    def analysis(self, tb):
        tb.ac(fstart=1, fstop=1e9)
```

## Integration with Optimization

Testbenches are the evaluation function for `circuit.optimize()`. The optimizer:

1. Exports the testbench as SPICE via `export_testbench_spice()`
2. Substitutes transistor W/L values computed from gm/Id lookup
3. Runs ngspice in batch mode
4. Parses `.meas` results from stdout
5. Feeds measurements back to the Bayesian optimizer

The testbench's `.meas` directive names must match the `name` fields in the optimization targets.
